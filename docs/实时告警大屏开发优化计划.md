### 1.2 现存缺陷与待优化项

| 编号 | 问题 | 影响 | 优先级 |
|---|---|---|---|
| OPT-01 | **无按场景筛选能力** | 大屏展示全量告警，16个场景混在一起，运维人员无法聚焦单场景 | P0 |
| OPT-02 | **无按厂部筛选能力** | 五大厂部（球团/烧结/炼钢/型钢/带钢）数据混合，各厂部值班员无法按域查看 | P0 |
| OPT-03 | **无告警级别快速过滤** | CRITICAL/WARNING/INFO 混列，紧急情况下需手动识别 | P1 |
| OPT-04 | **统计卡片不响应筛选** | 切换厂部后，4个统计数值仍为全量，与筛选上下文不一致 | P1 |
| OPT-05 | **趋势图与饼图不响应筛选** | 图表数据与筛选条件割裂 | P1 |
| OPT-06 | **场景列仅显示 scene_id** | 显示 `SCENE-SINTER-FIRE-001` 而非场景中文名，不直观 | P2 |
| OPT-07 | **告警表格无滚动动效** | 大量告警堆积时，旧数据不会自动滚动推入视野 | P2 |
| OPT-08 | **无各厂部告警热力分布图** | 无法一眼判断哪个厂部当前压力最大 | P2 |
| OPT-09 | **后端 `/alarms` 接口缺少 factory/scene_id 过滤参数** | 前端筛选只能在客户端过滤，数据量大时性能差 | P1 |
| OPT-10 | **后端 `/dashboard/overview` 缺少 factory 维度分组** | 无法获取单厂部的统计聚合数据 | P1 |

## 三、功能设计

### 3.1 筛选栏设计（新增，位于统计卡片行上方）

```
┌─────────────────────────────────────────────────────────────────────┐
│  🏭 厂部  [全部厂部 ▾]   📍 场景  [全部场景 ▾]   🔔 级别  [全部 ▾]  │
│                                              [↺ 重置筛选]           │
└─────────────────────────────────────────────────────────────────────┘
```

**厂部下拉选项**（固定5项 + 全部）：
```
全部厂部 / 球团厂(pellet) / 烧结厂(sintering) / 炼钢厂(steel) / 型钢厂(section) / 带钢厂(strip)
```

**场景下拉选项**（从后端动态加载，根据所选厂部联动过滤）：
```
全部场景 → 加载 GET /api/v1/scenes?factory={factory}&status=ACTIVE
单厂部 → 仅显示该厂部下的活跃场景
```

**级别过滤**（多选 Tag 形式，可组合）：
```
[严重 CRITICAL] [警告 WARNING] [信息 INFO]   ← 点击切换选中/取消
```

**联动规则**：
- 切换厂部 → 场景下拉重新加载 → 统计卡片/图表/告警表同步刷新
- 切换场景 → 统计卡片/图表/告警表同步刷新（厂部保持）
- 切换级别 → 仅告警表过滤（统计和图表不受影响，语义不同）
- 点击「重置筛选」→ 恢复默认全量展示

### 3.2 统计卡片行增强

在原有4张卡片基础上，筛选厂部后卡片数值切换为该厂部的子集统计：

| 卡片 | 无筛选 | 筛选厂部后 |
|---|---|---|
| 运行场景 | 全平台活跃场景数 | 该厂部活跃场景数 |
| 在线设备 | 全平台在线设备数 | 该厂部在线设备数 |
| 今日告警 | 全平台今日告警数 | 该厂部今日告警数 |
| 今日推理量 | 全平台今日推理量 | 该厂部今日推理量 |

新增第5张卡片（筛选场景后显示）：

| 卡片 | 说明 |
|---|---|
| 场景精度 | 该场景最近一次漂移监测精度值（来自 `drift_metric`） |

### 3.3 图表增强

**推理量趋势图**（现有，增加筛选参数）：
- 传入 `factory` / `scene_id` 参数，后端返回对应子集趋势
- 图例新增场景名称标注（多场景叠加折线，最多同时展示3条）

**告警级别分布饼图**（现有，增加筛选参数）：
- 传入 `factory` / `scene_id` 参数

**新增：各厂部告警热力柱图**（替换或并排展示）：
```
烧结厂  ████████████  23  ← 当日告警总数
炼钢厂  ██████        12
球团厂  ████           8
型钢厂  ██             4
带钢厂  █              2
```
无筛选时展示全厂对比，选中单厂部时高亮该厂部柱条。

### 3.4 实时告警表格增强

**新增列**：场景中文名（`sceneName`，后端返回）

**新增行为**：
- 告警列表以"跑马灯"方式每10s刷新时，新增告警从顶部淡入（CSS `animation: slideIn 0.4s`）
- 支持点击告警行跳转至管理后台告警详情页（新窗口）：`window.open('/alarm/{alarmId}')`

**表格列最终设计**：

| 列 | 宽度 | 说明 |
|---|---|---|
| 级别 | 90px | CRITICAL/WARNING/INFO 色块标签 |
| 厂部 | 90px | 中文厂部名 |
| 场景 | 160px | 场景中文名（新增） |
| 异常类型 | 180px | |
| 置信度 | 90px | 百分比 |
| 告警时间 | 150px | MM-DD HH:mm:ss |

// 筛选条件
const filterFactory  = ref<string>('')        // '' = 全部
const filterSceneId  = ref<string>('')        // '' = 全部
const filterLevels   = ref<string[]>(['CRITICAL', 'WARNING', 'INFO'])

// 场景列表（根据 filterFactory 联动加载）
const sceneOptions   = ref<{ sceneId: string; sceneName: string }[]>([])

// 计算属性：当前筛选条件是否有效（用于 UI 联动提示）
const hasFilter = computed(() =>
  filterFactory.value !== '' || filterSceneId.value !== ''
)

// 监听筛选条件变化，自动触发全量刷新
watch([filterFactory, filterSceneId], async () => {
  filterSceneId.value = ''           // 切换厂部时重置场景
  await loadSceneOptions()
  await Promise.all([fetchOverview(), fetchTrend(), fetchAlarms()])
})
watch(filterLevels, () => fetchAlarms())

### 4.2 API 层扩展

```typescript
// apps/monitor-screen/src/api/index.ts

export interface FilterParams {
  factory?: string     // pellet | sintering | steel | section | strip
  sceneId?: string
}

export const monitorApi = {
  // 原有接口，新增 factory/sceneId 参数透传
  getOverview: (params?: FilterParams) =>
    request.get<{ data: OverviewData }>('/dashboard/overview', { params }),

  getInferenceTrend: (days = 7, params?: FilterParams) =>
    request.get<{ data: TrendPoint[] }>('/dashboard/inference-trend', {
      params: { interval: '1d', days, ...params },
    }),

  // 新增：各厂部汇总统计（用于热力柱图）
  getFactorySummary: () =>
    request.get<{ data: FactorySummary[] }>('/dashboard/factory-summary'),

  getRecentAlarms: (size = 20, params?: FilterParams & { levels?: string[] }) =>
    request.get<{ data: { items: AlarmItem[] } }>('/alarms', {
      params: {
        page: 1, size,
        sort: 'created_at:desc',
        isSandbox: false,
        factory:  params?.factory  || undefined,
        scene_id: params?.sceneId  || undefined,
        levels:   params?.levels?.join(',') || undefined,
      },
    }),

  // 新增：获取活跃场景列表（用于场景下拉联动）
  getActiveScenes: (factory?: string) =>
    request.get<{ data: { items: SceneOption[] } }>('/scenes', {
      params: { factory, status: 'ACTIVE', page: 1, size: 100 },
    }),
}

// 新增类型
export interface FactorySummary {
  factory: string
  factoryName: string
  todayAlarms: number
  activeScenes: number
}

export interface SceneOption {
  sceneId: string
  sceneName: string
  factory: string
}
```

## 八、开发注意事项

### 8.1 筛选栏样式规范

```scss
// 大屏筛选栏：深色系，不破坏工业风格
.filter-bar {
  height: 48px;
  display: flex;
  align-items: center;
  gap: 24px;
  padding: 0 8px;
  border-bottom: 1px solid rgba(0, 120, 255, 0.15);
  flex-shrink: 0;

  // 下拉选择器统一暗色主题
  :deep(.el-select .el-input__wrapper) {
    background: rgba(5, 20, 50, 0.8);
    border-color: rgba(0, 120, 255, 0.3);
    box-shadow: none;
  }
}
```

> **禁止**：筛选栏不引入 Element Plus（大屏应用不依赖组件库，使用原生 `<select>` 或自定义下拉），
> 避免增加大屏包体积。可参考现有 `.level-badge` 样式风格实现自定义 Tag 多选。

### 8.2 场景名称映射

后端 `AlarmItem` 需包含 `sceneName` 字段（中文名），避免前端大屏维护场景 ID → 名称映射表。
如后端暂不支持，前端可在 `loadSceneOptions()` 时建立本地 `Map<sceneId, sceneName>` 缓存。

### 8.3 性能约束

- 场景下拉加载：`GET /scenes?status=ACTIVE` 单次请求，结果本地缓存，不随每次筛选重复请求
- 厂部热力图：`GET /dashboard/factory-summary` 与概览数据同频（30s）刷新
- 筛选条件变更防抖：`300ms debounce`，避免快速连续切换时触发多次请求

### 8.4 空数据处理

- 筛选后无告警数据：告警表显示「当前筛选条件下暂无告警」（区别于全量无数据）
- 筛选后场景下拉为空：显示「该厂部暂无活跃场景」
- 厂部热力柱图某厂部数据为 0：保留柱条（高度最小值 2px），不隐藏
