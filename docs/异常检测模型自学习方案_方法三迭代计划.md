# 异常检测模型自学习方案 — 方法三：先跑起来，再迭代标注

> **状态：计划在考虑中（Draft）**
> **日期**：2026-04-18
> **作者**：天柱·天镜项目组
> **背景**：仿真任务中上传录像时已支持正常/异常标签选项，基于此探索通过图像级标签自动训练推理模型的技术路线，避免前期大量人工框选标注的成本。

---

## 一、核心思路

传统路线要求对每一帧画精确 Bounding Box（边界框）才能训练检测模型，成本高、周期长。
本方案采用**图像级弱监督标签**驱动，分阶段迭代，以最低初始标注成本快速上线，再逐步精确化。

```
图像级标签（正常/异常/异常类型）
        ↓
  分类模型 / 异常检测模型（ONNX）
        ↓
  接入现有推理链路
        ↓
  收集误报/漏报 → 补充 BBox 标注 → 升级 YOLO 检测模型
```

---

## 二、两条技术路线对比

| 维度 | 图像级标签（本方案） | 传统 BBox 标注训练 |
|---|---|---|
| 标注成本 | 极低（只需正常/异常分类） | 高（每帧画框 + 类别） |
| 输出能力 | "这帧有问题" | "第 (x1,y1)-(x2,y2) 处有篦条断裂" |
| 定位精度 | 无或粗略热力图 | 精确像素级 |
| 训练数据量 | 少量异常即可起步 | 需足量各类缺陷样本 |
| 适合阶段 | 冷启动、快速验证 | 精确化阶段 |

---

## 三、整体路线图

```
阶段一（约 2 周）    阶段二（约 2 周）    阶段三（约 1 周）    阶段四（持续迭代）
数据采集与标注 ───→ 模型训练与导出 ───→ 接入推理链路 ───→ 迭代标注升级
上传 + 元数据入库    训练 + ONNX 导出    报警联动验收       精确化 → YOLO
```

---

## 四、阶段一：数据采集与标注（第 1–2 周）

**目标**：建立有标签的训练数据集

### 4.1 仿真任务上传 UI 扩展

```
现状：正常 / 异常 二选一
目标：正常 / 异常（若异常 → 追加"异常类型"下拉框）
异常类型枚举示例：篦条缺损、侧板偏移、铸坯裂纹、钢材表面缺陷、料面异常...
```

- 改造 `apps/admin-web/` 仿真任务上传页
- 异常类型从现有 16 个检测场景定义映射

### 4.2 新增录像元数据表

```sql
CREATE TABLE simulation_video (
    id           BIGSERIAL    PRIMARY KEY,
    scene_id     VARCHAR(64)  NOT NULL,
    video_url    VARCHAR(256) NOT NULL,     -- MinIO 地址
    label        VARCHAR(16)  NOT NULL,     -- NORMAL / ABNORMAL
    anomaly_type VARCHAR(64),               -- 异常类型，NORMAL 时为 NULL
    frame_count  INT,
    uploaded_by  VARCHAR(64),
    created_at   TIMESTAMPTZ  DEFAULT NOW()
);
```

### 4.3 帧提取 Worker

- 上传录像后，后台任务自动抽帧（建议每秒 2–3 帧），存入 MinIO
- 按路径归类存储，供训练脚本直接读取：

```
MinIO bucket: tianjing-datasets
└── {scene_id}/
    ├── normal/
    │   └── frame_001.jpg, frame_002.jpg ...
    └── abnormal/
        ├── 篦条缺损/
        └── 铸坯裂纹/
```

- 可复用 `inference/recording-replay-service` 中已有的 OpenCV 抽帧逻辑

**阶段一交付物**：MinIO 中 normal / abnormal 两类图像目录，每场景至少 200 帧正常 + 100 帧异常（每类）

## 五、阶段二：模型训练与导出（第 3–4 周）

**目标**：训练出可用的 ONNX 分类/异常检测模型

### 5.1 模型选型

| 适用情形 | 推荐方法 | 说明 |
|---|---|---|
| 只有正常帧、异常罕见 | **PatchCore**（anomalib） | 零异常标注，只用正常帧训练 |
| 有正常 + 多类异常帧 | **ResNet-50 分类器** | 图像级标签直接训，支持多类输出 |

> **第一版本选用 ResNet-50 分类器**，因已有正常/异常+类别标签，入手快；
> 同步保留 PatchCore 作为兜底，检测未见过的新型异常。

### 5.2 训练脚本

新增：`training/train_jobs/anomaly_classify/train.py`

```
核心流程：
1. 从 MinIO 读取 normal/ 和 abnormal/{type}/ 帧
2. ResNet-50 fine-tune（冻结前 4 层，只训最后 FC + 任务头）
3. MLflow 记录 accuracy / F1 / confusion matrix
4. 导出 ONNX（opset 17，input shape: [B, 3, 640, 640]）
5. 推送模型至 Harbor: tianjing-models-staging
```

### 5.3 推理输出适配

分类模型无 BBox，需适配现有 `InferOutput` schema（下游 alarm-judge-service 无需改动）：

```python
# 分类结果 → InferOutput detections[] 格式
if anomaly_score > threshold:
    detections = [Detection(
        class_name=anomaly_type_name,   # 如 "篦条缺损"
        confidence=anomaly_score,
        bbox={"x1": 0, "y1": 0,         # 全图 BBox，表示"整帧异常"
              "x2": W,  "y2": H},
    )]
else:
    detections = []
```

新增 plugin_id：`CLASSIFY-ANOMALY-V1`

**阶段二交付物**：MLflow 中注册的 ONNX 模型，精度报告（F1 ≥ 0.85 方可进入阶段三）

## 六、阶段三：接入推理链路与报警（第 5 周）

**目标**：模型上线，录像回放能触发真实报警

### 6.1 改动范围（最小化）

| 服务 | 改动内容 |
|---|---|
| `gpu-infer-service` | 加载新 ONNX 分类模型（框架已就绪，注册 CLASSIFY-ANOMALY-V1 插件） |
| `scene-config-service` | 在场景配置中绑定新模型版本 + plugin_id |
| `route-dispatch-service` | **无需改动** |
| `alarm-judge-service` | **无需改动**（全图 BBox 方案完全兼容） |
| `apps/admin-web/` | **无需改动** |

### 6.2 验收标准

```
✅ 播放"异常"录像 → 推理 confidence ≥ 0.8 且连续 2 帧 → 触发 WARNING 级报警
✅ 播放"正常"录像 → 连续 30 秒无告警
✅ 今日推理量 + 告警量在监控大屏正常显示
✅ Sandbox 拦截器正常工作（仿真结果不触达 MQTT / 生产告警）
```

## 七、阶段四：迭代标注，逐步精确化（持续）

上线后收集误报/漏报数据，逐步升级模型精度与定位能力：

```
第一轮（上线后 2 周）
├── 收集误报帧（模型说异常但实际正常）
├── 收集漏报帧（模型未报但人工复核确认异常）
└── 人工对这些帧补画 BBox → 加入训练集

第二轮（上线后 1 个月）
├── 每类异常积累 ≥ 500 帧 + BBox
├── 基于 MMDetection 训练 YOLOv9 检测头
└── 替换 ONNX 分类模型 → YOLO 检测模型，精确定位生效

长期
└── drift-monitor-service 自动监测精度下降 → 触发重训练流水线
```

---

## 八、关键风险与应对

| 风险 | 应对措施 |
|---|---|
| 异常样本太少（< 100 帧/类） | 改用 PatchCore，只需正常帧即可训练 |
| 误报率高导致运维投诉 | 告警连续帧数由 2 调至 3，置信度阈值提高至 0.88 |
| 全图 BBox 导致第四阶段回标浪费 | 回标时图像已存在，只需补画框，无需重新采集 |
| 模型只识别训练过的异常类型 | 并行部署 PatchCore 作兜底，检测未见类型 |
| 训练数据分布不均（正常帧 >> 异常帧） | 训练时对异常类别进行过采样（WeightedRandomSampler） |
