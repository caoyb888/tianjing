# 训练作业过程 — 烧结看火场景全链路操作说明

> **适用范围**：Sprint 3 S3-13 训练链路端到端验证（CPU 模式，仅验证流程，不要求精度）
> **场景**：烧结看火料面状态检测（SCENE-SINTER-FIRE-001）
> **编制日期**：2026-04-05

---

## 整体流程

```
录像文件（mp4）
    ↓ ① 抽帧
帧图像（jpg）
    ↓ ② 标注（Label Studio）
COCO 标注文件（annotations.json）
    ↓ ③ 打包上传 MinIO
dataset.zip（images/ + annotations.json）
    ↓ ④ 提交训练作业
管理后台 → 训练管理 → 提交训练作业
    ↓ ⑤ 训练容器自动启动（docker run）
PENDING → RUNNING → COMPLETED
    ↓ ⑥ 自动注册模型版本
训练作业详情页显示 MV-XXXXXXXX → 点击进入审核流程
```

---

## 第一步：准备数据集 zip 包

训练脚本从 MinIO 下载固定格式的 zip 包，路径规则：

```
Bucket: tianjing-datasets
Key:    DV-MATERIAL-LEVEL-V1-1/dataset.zip
```

**zip 内部结构要求**（`training/train_jobs/train_yolo_cpu.py` 约定）：

```
dataset.zip
├── images/
│   ├── frame_000001.jpg
│   ├── frame_000002.jpg
│   └── ...（至少 50 张，Sprint 3 精度不要求）
└── annotations.json   ← COCO 格式标注文件
```

> **Sprint 3 说明**：CPU 训练仅验证链路流程，不要求精度达标。
> 如暂无完整标注数据，可使用空标注文件运行（见下方步骤）。

---

## 第二步：从录像抽帧并上传 MinIO

```bash
# ① 抽帧（每秒 1 帧，取前 300 帧即可）
mkdir -p /tmp/sinter-frames
ffmpeg -i /path/to/看火录像.mp4 -vf fps=1 -frames:v 300 \
  /tmp/sinter-frames/frame_%06d.jpg

# ② 生成最小可用的 COCO 标注文件（Sprint 3 流程验证用）
python3 - <<'EOF'
import json, os
from pathlib import Path

images = []
for i, f in enumerate(sorted(Path("/tmp/sinter-frames").glob("*.jpg")), 1):
    images.append({"id": i, "file_name": f.name, "width": 1920, "height": 1080})

coco = {
    "images": images,
    "annotations": [],       # Sprint 3：空标注，仅验证链路
    "categories": [{"id": 1, "name": "material_level"}]
}
with open("/tmp/sinter-frames/annotations.json", "w") as fp:
    json.dump(coco, fp)
print(f"生成 {len(images)} 张图像的标注文件")
EOF

# ③ 打包（确保 zip 内部结构为 images/ + annotations.json）
mkdir -p /tmp/pkg/images
cp /tmp/sinter-frames/*.jpg /tmp/pkg/images/
cp /tmp/sinter-frames/annotations.json /tmp/pkg/
cd /tmp/pkg && zip -r /tmp/dataset.zip images/ annotations.json

# ④ 上传到 MinIO
docker run --rm --network host --entrypoint sh minio/mc:latest -c "
  mc alias set local http://localhost:9000 minioadmin minioadmin123 --quiet
  mc cp /tmp/dataset.zip local/tianjing-datasets/DV-MATERIAL-LEVEL-V1-1/dataset.zip
  mc ls local/tianjing-datasets/DV-MATERIAL-LEVEL-V1-1/
"
```

## 第三步：管理后台提交训练作业

打开 `http://10.30.10.49:5173/training/jobs`，点击「提交训练作业」：

| 字段 | 填写值 | 说明 |
|---|---|---|
| **插件 ID** | `HEAD-MATERIAL-LEVEL-V1`（料面高度估计头） | 下拉选择，烧结看火对应的任务头 |
| **数据集版本** | `DS-MATERIAL-LEVEL-001 · v1.1`（`DV-MATERIAL-LEVEL-V1-1`） | 未冻结版本，刚才上传的目标 |
| **训练轮次** | `10` | Sprint 3 CPU 模式，仅验证流程 |
| **GPU 数量** | `1` | CPU 模式下此字段被训练脚本忽略 |

---

## 第四步：确认训练容器启动

提交后，`DockerTrainJobLauncher` 会在 Server 1 本地自动启动训练容器：

```bash
# 确认容器正在运行
docker ps | grep "train-TJ-"

# 实时查看训练日志
docker logs -f $(docker ps --filter "name=train-TJ-" -q)
```

状态变化（管理后台训练作业详情页每 5 秒自动轮询）：

```
PENDING  →  RUNNING（训练容器启动后回调）
         →  COMPLETED（10 epoch 完成，CPU 约 30~60 分钟）
```

---

## 第五步：确认训练完成与模型注册

训练完成后，脚本自动：
1. 将模型导出为 ONNX 格式并上传至 `tianjing-models-staging/{PLUGIN_ID}/{JOB_ID}/best.onnx`
2. 在 MLflow 注册模型版本（可在 `http://Server1:5000` 查看）
3. 回调平台接口写入 `model_version_id`

训练作业详情页顶部将出现绿色横幅：

```
训练完成，已自动注册模型版本 MV-XXXXXXXX  [进入模型审核流程]
```

点击按钮跳转至模型详情页，提交审核，由 MODEL_REVIEWER 角色审批通过后，模型状态变为 `PRODUCTION`。

## 关键前提确认

运行前请确认以下两项：

```bash
# 1. 训练镜像是否已构建
docker images | grep "yolo-trainer"

# 如未构建，先执行：
cd /home/xintong/tianjing/training/train_jobs
docker build -t tianjing/yolo-trainer:cpu-latest .

# 2. MLflow 是否可达
curl -s --noproxy "*" http://localhost:5000/health
```

> **MLflow 不可达时的降级方案**：训练脚本会跳过自动注册模型版本，需训练完成后手动在
> 「模型版本管理」页面填写信息提交。自动注册可列入 Sprint 4 补全。
