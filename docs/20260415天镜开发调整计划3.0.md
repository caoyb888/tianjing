## Sprint 3：烧结看火场景 GPU 推理端到端全功能验证（05/01 – 05/10）

### Sprint 目标

以**烧结看火场景**为唯一对象，使用实验室录像 + **V100 GPU 本地推理**，验证系统全部功能闭环：
**录像注入 → GPU 实时推理（≤20ms）→ 告警触发 → Sandbox 对比 → 模型审核 → 人工反馈 → 漂移监测。**

**本 Sprint 与 V2.0 Sprint 3 的核心差异**：推理引擎从"云端代理（>1s）"升级为"本地 V100 GPU（≤20ms）"，
验收演示时甲方可直观感受到实时推理效果，而非依赖网络的云端调用。

**烧结看火场景基础信息**：
- `scene_id`: `SCENE-SINTER-FIRE-001`
- 场景类型：工艺参数检测（点火强度 / 料面状态）
- 算法插件：**`LOCAL-GPU-YOLO-V1`**（V100 本地推理，替代 V2.0 的云端代理）
- 数据来源：实验室录像（30+ 分钟，含正常 / 点火偏弱 / 料面异常片段）
- 推理精度目标：mAP ≥ 0.80（基础预训练权重，Sprint 4 用标注数据微调至 ≥0.85）

1. **cuda-compat 目录冲突**：`nvidia/cuda:12.3.2-base` 镜像内 `/usr/local/cuda/compat/libcuda.so.545` 覆盖了宿主机注入的 `libcuda.so.570`，导致 `cudaSetDevice` 返回 Error 999。解法：Dockerfile 构建阶段末尾执行 `rm -rf /usr/local/cuda/compat/`，使容器直接使用宿主机驱动。

2. **cuDNN 版本选型**：NVIDIA apt 仓库默认最新版 `libcudnn9-cuda-12=9.21.0.82-1` 为 CUDA 12.9 编译，与 CUDA 12.3 Runtime + V100 SM70（Volta 架构）不兼容，推理时抛出 `CUDNN_STATUS_EXECUTION_FAILED_CUDART (5003)`。解法：固定版本为 `libcudnn9-cuda-12=9.2.1.18-1`（CUDA 12.3 兼容，V100 验证通过）。

3. **inference_time_ms 计时范围修正**：原代码将图像 base64 解码 + letterbox 预处理 + ONNX 推理 + NMS 后处理全部纳入计时（导致 P95≈28ms 看似未达标）。按 CLAUDE.md §5.2 规范，`inference_time_ms` 仅计量 `session.run()` 阶段，修正后 P95 降至 **9.6ms**。
