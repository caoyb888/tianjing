## 6. 本地开发快速启动

### 前端

```bash
cd apps/admin-web
npm install
npm run dev          # → http://localhost:5173
```

> `vite.config.ts` 中 `/api` 代理目标为 `http://localhost:8079`。本地开发时可按需改为具体服务端口，或在 8079 起一个 Nginx 做聚合反代。

### 后端微服务（以 auth-service 为例）

```bash
cd services/auth-service
./mvnw spring-boot:run    # → http://localhost:8081
```

### 中间件（Docker 单机启动）

```bash
# PostgreSQL
docker run -d --name pg -p 5432:5432 \
  -e POSTGRES_PASSWORD=tianjing \
  postgres:16

# Redis
docker run -d --name redis -p 6379:6379 redis:7

# Kafka（KRaft 模式，无需 ZooKeeper）
docker run -d --name kafka -p 9092:9092 \
  -e KAFKA_CFG_NODE_ID=1 \
  -e KAFKA_CFG_PROCESS_ROLES=controller,broker \
  -e KAFKA_CFG_LISTENERS=PLAINTEXT://:9092,CONTROLLER://:9093 \
  -e KAFKA_CFG_ADVERTISED_LISTENERS=PLAINTEXT://localhost:9092 \
  -e KAFKA_CFG_CONTROLLER_QUORUM_VOTERS=1@localhost:9093 \
  bitnami/kafka:3.6

# Nacos（单机模式）
docker run -d --name nacos -p 8848:8848 -p 9848:9848 \
  -e MODE=standalone \
  nacos/nacos-server:v2.3.0

# MinIO
docker run -d --name minio -p 9000:9000 -p 9001:9001 \
  -e MINIO_ROOT_USER=tianjing \
  -e MINIO_ROOT_PASSWORD=tianjing123 \
  minio/minio server /data --console-address ":9001"

# TDengine
docker run -d --name tdengine -p 6030:6030 -p 6041:6041 \
  tdengine/tdengine:3.0.7.1
```

## 7. 端口冲突速查

| 常见冲突场景 | 解决办法 |
|---|---|
| **8080 / 8088** | 开发服务器保留，本系统禁止使用，见 CLAUDE.md P1-8 |
| 5173 被占用 | 修改 `vite.config.ts` → `server.port` |
| 8079 被占用 | 修改 `vite.config.ts` → `proxy['/api'].target`，同步修改 Nginx 监听端口 |
| 5432 被本地 PostgreSQL 占用 | Docker 映射改为 `-p 5433:5432`，同步修改各服务 `application.yml` 中 DB URL |
| 6379 被本地 Redis 占用 | 设置环境变量 `TIANJING_REDIS_PORT=6380` |
| 9092 被其他 Kafka 占用 | 设置环境变量 `TIANJING_KAFKA_BOOTSTRAP_SERVERS=localhost:9093` |
