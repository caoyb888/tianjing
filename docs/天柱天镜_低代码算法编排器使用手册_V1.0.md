# 低代码算法编排器使用手册

> **版本**：V1.0 · **日期**：2026-04-04
> **适用对象**：具有 `SCENE_EDITOR` 角色的场景配置人员
> **入口路径**：管理后台 → 场景管理 → 选中场景 → 算法编排

---

## 1. 概述

低代码算法编排器是天柱·天镜平台的核心配置工具，允许用户通过**拖拽节点、连线**的方式，将原子算法组件组合成完整的视觉推理流水线，无需修改代码。

编排完成后保存为工作流 JSON，经**发布**操作下发至路由分发服务（route-dispatch-service），推理链路实时热加载生效。

## 3. 节点类型说明

编排器提供 7 种节点，每种节点有固定职责，**数据流向必须从左至右**（每个节点左侧为输入端口，右侧为输出端口）。

| 节点类型 | 颜色 | 说明 | 对应后端类型 |
|----------|------|------|-------------|
| 视频源 | 🟢 绿色 | 推理帧数据来源，每个流水线必须有且仅有一个 | `FRAME_IN` |
| 预处理 | 🔵 蓝色 | 图像增强：去雾、裁剪、亮度调整等，可串联多个 | `PREPROCESS` |
| 推理插件 | 🟣 紫色 | 调用原子算法执行推理（检测/分割/分类/测量） | `DETECT` / `SEGMENT` / `CLASSIFY` / `MEASURE` |
| 逻辑门 | 🟠 橙色 | 条件过滤：对推理结果进行阈值、帧数、类别过滤 | 内嵌于 `ALARM_OUT` 前置逻辑 |
| 告警动作 | 🔴 红色 | 触发平台告警，每个流水线必须有且仅有一个 | `ALARM_OUT` |
| 存储动作 | 🩵 青色 | 将异常帧写入 MinIO 对象存储 | `ALARM_OUT` 附属 |
| MQTT 推送 | 💗 粉色 | 向工业互联网平台推送告警消息（QoS=1） | `ALARM_OUT` 附属 |

> **强制规则**：工作流保存时后端会校验——`视频源` 节点必须恰好 1 个，`告警动作` 节点必须恰好 1 个，否则保存失败。

## 4. 操作步骤

### 4.1 进入编排器

1. 登录管理后台（需 `SCENE_EDITOR` 角色）
2. 左侧菜单 → **场景管理**
3. 找到目标场景，点击操作列中的**算法编排**按钮
4. 编排器自动加载该场景已有的工作流（若无则为空白画布）

### 4.2 添加节点

从左侧**节点面板**将节点拖拽至中央画布的目标位置，松手即可放置。

```
拖拽步骤：
  1. 鼠标悬停在节点面板的节点上（光标变为 ✋）
  2. 按住鼠标左键开始拖拽
  3. 拖至画布中目标位置松开
  → 节点以默认配置出现在画布上
```

### 4.3 连接节点

将上游节点的**右侧圆点**（输出端口）拖至下游节点的**左侧圆点**（输入端口），松开即建立连接。

连接线会以带动画的平滑曲线显示，表示数据流向。

**标准流水线结构示例：**

```
视频源  →  预处理  →  推理插件  →  逻辑门  →  告警动作
                                              ↓
                                          MQTT 推送 / 存储动作
```

### 4.4 配置节点属性

点击画布中的任意节点，右侧**属性面板**展开，可配置以下参数：

#### 通用属性（所有节点）

| 属性 | 说明 |
|------|------|
| 节点标签 | 在画布上显示的名称，便于识别，不影响执行逻辑 |

#### 推理插件节点

| 属性 | 说明 | 示例 |
|------|------|------|
| 插件 ID | 调用的原子算法插件标识符 | `ATOM-DETECT-YOLO-V1` |
| 置信阈值 | 推理结果低于此值时忽略该检测框，范围 0.00–1.00 | `0.85` |

> 插件 ID 可在**管理后台 → 算法插件**页面中查询所有已注册插件。

#### 告警动作节点

| 属性 | 说明 | 可选值 |
|------|------|--------|
| 告警级别 | 触发告警的严重程度 | `CRITICAL` / `WARNING` / `INFO` |
| 确认帧数 | 连续多少帧检测到异常才真正触发告警，防止误报 | 建议 2–5 帧 |

#### 逻辑门节点

| 属性 | 说明 |
|------|------|
| 置信度阈值 | 只有置信度超过阈值的结果通过 |
| 连续帧计数 | 连续 N 帧均检测到才通过 |
| 类别过滤 | 只允许指定类别的检测结果通过 |

#### MQTT 推送节点

| 属性 | 说明 | 默认值 |
|------|------|--------|
| Topic | 消息发布的 MQTT Topic | `iiot/tianjing/alarm/{场景ID}` |

### 4.5 删除节点

点击节点后，在右侧属性面板底部点击**删除节点**按钮，与该节点相连的边也会同步移除。

### 4.6 保存与发布

| 操作 | 按钮位置 | 说明 |
|------|---------|------|
| **保存编排** | 顶部工具栏 | 保存为 `DRAFT`（草稿）状态，不影响线上推理 |
| **重新加载** | 顶部工具栏 | 放弃当前未保存的改动，从数据库重新加载 |
| **发布**（API） | 调用 `POST /api/v1/workflows/{id}/publish` | 将草稿变更为 `PUBLISHED`，同步至 Nacos 配置中心，路由服务热加载 |

> 当前管理后台的"保存编排"按钮直接调用场景配置接口（`sceneApi.update`），将 workflowJson 存入 `scene_config.config_json`。独立的工作流发布按钮可通过 API 或后续版本迭代补充。

## 5. 工作流状态流转

```
新建（编排器拖拽）
       ↓ 保存编排
    DRAFT（草稿）
       ↓ 发布
   PUBLISHED（已发布，线上生效）
       ↓ 归档
   ARCHIVED（已归档，不可直接删除）
```

| 状态 | 说明 | 可进行的操作 |
|------|------|-------------|
| `DRAFT` | 草稿，未影响线上 | 编辑、发布、删除 |
| `PUBLISHED` | 已发布，推理服务正在使用 | 归档（不可直接删除） |
| `ARCHIVED` | 已归档，历史记录 | 只读 |

## 7. 注意事项

1. **Sandbox 标识不影响编排逻辑**：编排器配置的推理规则在生产流量和 Sandbox 镜像流量中完全一致，`is_sandbox` 由平台底层透传，不在编排器层面区分。

2. **不支持 `eval` 执行自定义代码**：逻辑门只提供有限的条件类型，禁止通过任何方式在编排器中执行用户输入的任意代码（安全红线 P0）。

3. **已发布的工作流不可直接删除**：需先归档（`ARCHIVED`），再在历史记录中操作。

4. **编排器与告警规则是独立配置**：编排器负责推理流水线逻辑（选哪个模型、如何串联），告警触发阈值和推送渠道在**告警规则**页面单独配置。

5. **插件 ID 需提前注册**：使用推理插件节点前，请确认目标插件已在算法插件管理页面中注册且状态为 `ACTIVE`。
