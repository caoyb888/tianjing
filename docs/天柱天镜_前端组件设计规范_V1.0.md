### 1.2 命名规范速查

| 元素 | 规范 | 示例 |

| Pinia Store 文件 | `<domain>.ts`，导出 `use<Domain>Store` | `scene.ts` → `useSceneStore` |

### 2.3 路由 Meta 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `requiresAuth` | `boolean` | `false` 则跳过 Token 校验（默认 `true`） |
| `title` | `string` | 页面标题，用于面包屑和 `<title>` |
| `roles` | `Role[]` | 可访问的角色列表；空数组 `[]` 表示所有已登录用户可访问 |

## 3. 组件层级划分

### 3.1 组件分层模型

```
components/
├── layout/          【布局组件】全局唯一，承载整体框架
├── common/          【通用基础组件】无业务语义，高度可复用
├── business/        【业务组件】携带业务语义，跨页面复用
└── charts/          【图表组件】数据可视化，封装 ECharts
```

### 3.2 布局组件（`components/layout/`）

| 组件 | 职责 |
|------|------|
| `AppLayout.vue` | 整体外框：侧边栏 + 顶栏 + 内容区，路由 `<RouterView>` 出口 |
| `AppSidebar.vue` | 导航菜单，基于 `usePermission` 动态过滤无权限菜单项 |
| `AppHeader.vue` | 顶部栏：面包屑、用户头像、告警铃铛（SSE 实时数）、全屏切换 |
| `AppBreadcrumb.vue` | 基于当前路由 `meta.title` 链自动生成面包屑 |
| `AppFooter.vue` | 版本号、公司信息（可选） |

**`AppLayout.vue` 结构示意：**

```vue
<template>
  <el-container class="app-layout">
    <AppSidebar :collapsed="layout.sidebarCollapsed" />
    <el-container direction="vertical">
      <AppHeader />
      <el-main>
        <RouterView v-slot="{ Component }">
          <Transition name="fade-slide" mode="out-in">
            <KeepAlive :include="keepAliveRoutes">
              <component :is="Component" />
            </KeepAlive>
          </Transition>
        </RouterView>
      </el-main>
    </el-container>
  </el-container>
</template>
```

> `keepAliveRoutes` 包含：`SCENE_LIST`、`ALARM_LIST`、`MODEL_LIST`（列表页保留滚动位置和筛选条件）。

### 3.3 通用基础组件（`components/common/`）

| 组件 | Props 关键字段 | 用途 |
|------|-------------|------|
| `StatusBadge.vue` | `status: string`, `colorMap: Record<string, string>` | 通用状态标签（支持场景状态/模型状态/告警级别） |

### 4.1 Store 职责边界

### 4.2 authStore 详细设计

```typescript
// src/stores/auth.ts
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'
import type { UserInfo } from '@/types/api'
import { Role } from '@/types/enums'

export const useAuthStore = defineStore('auth', () => {
  // ── State ──────────────────────────────────────────────────
  const token = ref<string | null>(null)      // 内存存储，不持久化
  const user = ref<UserInfo | null>(null)
  const tokenExpiresAt = ref<number | null>(null)  // Unix timestamp (ms)

  // ── Getters ────────────────────────────────────────────────
  const isAuthenticated = computed(() => !!token.value && !isTokenExpired.value)
  const userRoles = computed<Role[]>(() => user.value?.roles ?? [])
  const isAdmin = computed(() => userRoles.value.includes(Role.ADMIN))

  const isTokenExpired = computed(() => {
    if (!tokenExpiresAt.value) return true
    // 提前5分钟判定为即将过期，触发静默刷新
    return Date.now() >= tokenExpiresAt.value - 5 * 60 * 1000
  })

  // ── Actions ────────────────────────────────────────────────
  function setAuth(accessToken: string, expiresIn: number, userInfo: UserInfo) {
    token.value = accessToken
    user.value = userInfo
    tokenExpiresAt.value = Date.now() + expiresIn * 1000
  }

  function clearAuth() {
    token.value = null
    user.value = null
    tokenExpiresAt.value = null
  }

  function hasRole(roles: Role[]): boolean {
    if (roles.length === 0) return true  // 空数组 = 所有角色可访问
    return roles.some(r => userRoles.value.includes(r))
  }

  return { token, user, isAuthenticated, userRoles, isAdmin, isTokenExpired,
           setAuth, clearAuth, hasRole }
})
```

### 4.3 alarmStore 详细设计

```typescript
// src/stores/alarm.ts
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'
import type { AlarmRecord } from '@/types/api'

const MAX_REALTIME_ALARMS = 50  // 顶栏最多缓存50条实时告警

export const useAlarmStore = defineStore('alarm', () => {
  const realtimeAlarms = ref<AlarmRecord[]>([])
  const unreadCount = ref(0)
  const sseConnected = ref(false)

  const criticalAlarms = computed(() =>
    realtimeAlarms.value.filter(a => a.alarm_level === 'CRITICAL')
  )

  function pushAlarm(alarm: AlarmRecord) {
    realtimeAlarms.value.unshift(alarm)
    if (realtimeAlarms.value.length > MAX_REALTIME_ALARMS) {
      realtimeAlarms.value.pop()
    }
    unreadCount.value++
  }

  function markAllRead() {
    unreadCount.value = 0
  }

  return { realtimeAlarms, unreadCount, sseConnected, criticalAlarms,
           pushAlarm, markAllRead }
})
```

### 5.2 全局路由守卫

```typescript
// src/router/guards.ts
import { Router } from 'vue-router'
import { useAuthStore } from '@/stores/auth'
import { refreshToken } from '@/api/auth'

export function setupRouterGuards(router: Router) {
  router.beforeEach(async (to, _from, next) => {
    const auth = useAuthStore()

    // ① 无需鉴权的页面（登录页、错误页）直接放行
    if (to.meta.requiresAuth === false) {
      // 已登录用户访问登录页，重定向到首页
      if (auth.isAuthenticated && to.name === 'LOGIN') return next('/dashboard')
      return next()
    }

    // ② 未登录：重定向到登录页，记录来源路由
    if (!auth.token) {
      return next({ name: 'LOGIN', query: { redirect: to.fullPath } })
    }

    // ③ Token 即将过期：静默刷新（不打断用户操作）
    if (auth.isTokenExpired) {
      try {
        await refreshToken()
      } catch {
        auth.clearAuth()
        return next({ name: 'LOGIN', query: { redirect: to.fullPath } })
      }
    }

    // ④ 角色权限校验
    const requiredRoles = (to.meta.roles as string[]) ?? []
    if (!auth.hasRole(requiredRoles)) {
      return next({ name: 'FORBIDDEN' })
    }

    // ⑤ 设置页面标题
    document.title = `${to.meta.title ?? '天柱天镜'} · 天镜平台`

    next()
  })
}
```

### 5.3 `usePermission` Composable

封装权限判断逻辑，在组件中统一调用，禁止直接读取 Store 进行权限判断：

```typescript
// src/composables/usePermission.ts
import { computed } from 'vue'
import { useAuthStore } from '@/stores/auth'
import { Role } from '@/types/enums'

export function usePermission() {
  const auth = useAuthStore()

  /**
   * 判断当前用户是否拥有指定角色之一
   * @param roles 空数组表示所有登录用户均可访问
   */
  const can = (roles: Role[]): boolean => auth.hasRole(roles)

  // 预定义常用权限语义，减少硬编码
  const canEditScene    = computed(() => can([Role.SCENE_EDITOR, Role.ADMIN]))
  const canReviewModel  = computed(() => can([Role.MODEL_REVIEWER, Role.ADMIN]))
  const canOperateSandbox = computed(() => can([Role.SANDBOX_OPERATOR, Role.ADMIN]))
  const canManageSystem = computed(() => can([Role.ADMIN]))

  return { can, canEditScene, canReviewModel, canOperateSandbox, canManageSystem }
}
```

### 5.4 `v-permission` 自定义指令

对于需要在模板中批量控制 DOM 可见性的场景，使用自定义指令：

```typescript
// src/directives/permission.ts
import { Directive } from 'vue'
import { useAuthStore } from '@/stores/auth'
import { Role } from '@/types/enums'

/**
 * 用法：
 *   v-permission="[Role.ADMIN]"              无权限时隐藏元素
 *   v-permission.disable="[Role.ADMIN]"      无权限时禁用元素（保留占位）
 */
export const vPermission: Directive<HTMLElement, Role[]> = {
  mounted(el, binding) {
    const auth = useAuthStore()
    const roles: Role[] = binding.value ?? []
    const hasPermission = auth.hasRole(roles)

    if (!hasPermission) {
      if (binding.modifiers.disable) {
        el.setAttribute('disabled', 'true')
        el.style.opacity = '0.4'
        el.style.pointerEvents = 'none'
      } else {
        el.style.display = 'none'
      }
    }
  },
}

// main.ts 中注册
// app.directive('permission', vPermission)
```

## 6. API 请求层封装

### 6.1 Axios 实例与拦截器

```typescript
// src/api/_http.ts
import axios from 'axios'
import { v4 as uuidv4 } from 'uuid'
import { useAuthStore } from '@/stores/auth'
import { ElMessage } from 'element-plus'
import router from '@/router'

const http = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL,  // 来自环境变量，不硬编码
  timeout: 10_000,
  headers: { 'Content-Type': 'application/json' },
})

// ── 请求拦截器 ─────────────────────────────────────────────────
http.interceptors.request.use(config => {
  const auth = useAuthStore()
  if (auth.token) {
    config.headers.Authorization = `Bearer ${auth.token}`
  }
  config.headers['X-Request-ID'] = uuidv4()
  return config
})

// ── 响应拦截器 ─────────────────────────────────────────────────
http.interceptors.response.use(
  response => {
    const { code, message, data } = response.data
    if (code !== 0) {
      // 业务错误：统一弹出错误提示，并将 trace_id 附在消息中便于排查
      const traceId = response.data.trace_id
      ElMessage.error(`${message}（trace: ${traceId?.slice(0, 8)}）`)
      return Promise.reject({ code, message, traceId })
    }
    return data  // 调用方直接拿到 data，无需 .data.data
  },
  error => {
    if (error.response?.status === 401) {
      useAuthStore().clearAuth()
      router.push({ name: 'LOGIN' })
      ElMessage.error('登录已过期，请重新登录')
    } else if (error.response?.status === 403) {
      router.push({ name: 'FORBIDDEN' })
    } else {
      ElMessage.error('网络请求失败，请检查网络连接')
    }
    return Promise.reject(error)
  }
)

export default http
```

## 7. 实时通信约定

### 7.1 SSE（Server-Sent Events）封装

看板大屏实时告警使用 `GET /dashboard/alarms/realtime` SSE 接口：

```typescript
// src/composables/useRealtime.ts
import { onMounted, onUnmounted, ref } from 'vue'
import { useAuthStore } from '@/stores/auth'
import { useAlarmStore } from '@/stores/alarm'
import type { AlarmRecord } from '@/types/api'

export function useRealtimeAlarms() {
  const auth = useAuthStore()
  const alarmStore = useAlarmStore()
  let eventSource: EventSource | null = null

  function connect() {
    const url = new URL(`${import.meta.env.VITE_API_BASE_URL}/dashboard/alarms/realtime`)
    url.searchParams.set('token', auth.token!)  // SSE 不支持 Header，通过 Query 传递 Token

    eventSource = new EventSource(url.toString())
    alarmStore.sseConnected = true

    eventSource.onmessage = (event) => {
      const alarm: AlarmRecord = JSON.parse(event.data)
      alarmStore.pushAlarm(alarm)
    }

    eventSource.onerror = () => {
      alarmStore.sseConnected = false
      // 指数退避重连（最大等待 30s）
      setTimeout(connect, Math.min(reconnectDelay.value, 30_000))
      reconnectDelay.value = Math.min(reconnectDelay.value * 2, 30_000)
    }
  }

  const reconnectDelay = ref(1000)

  onMounted(connect)
  onUnmounted(() => {
    eventSource?.close()
    alarmStore.sseConnected = false
  })

  return { connected: alarmStore.sseConnected }
}
```

| 实时监控大屏 | WebRTC | FLV | < 500ms | MinIO 截帧静态图（5fps 刷新） |
| 在线标定工具 | FLV（HTTP-FLV） | HLS | 1-3s 可接受 | MinIO 截帧静态图（手动刷新） |
| 告警回放 / 仿真结果 | HLS | — | 无要求 | 同 Sprint 5（HLS 播放 MinIO 录像） |

### 8.2 VideoPlayer 组件规范

```typescript
// src/components/business/VideoPlayer.vue Props 定义
interface Props {
  deviceCode: string
  protocol?: 'webrtc' | 'flv' | 'hls'   // 不传则由组件自动选择
  quality?: 'original' | 'low'
  showControls?: boolean                  // 标定模式不显示进度条
  overlayCanvas?: boolean                 // 是否开启 Canvas 叠加（ROI 标定/检测框展示）
}
```

**组件内部流程（Sprint 5 实时流模式，GPU 到位后启用）：**

```
mount → 调用 GET /devices/:deviceCode/live-stream 获取流地址
      → 根据 protocol 选择播放器：
          WebRTC → RTCPeerConnection API（原生）
          FLV   → mpegts.js 库
          HLS   → hls.js 库
      → 建立播放
      → 1小时前30分钟自动续签（重新调用接口获取新地址）
unmount → 调用流媒体服务释放 session_token → 关闭播放器连接
```

**★V1.1 实验室阶段（Sprint 0-4）行为：**

```
mount → 调用 GET /devices/:deviceCode/live-stream
      → 后端返回 HTTP 501（Not Implemented）
      → 组件检测到 501：
          IF 有 minioFrameUrl prop（标定工具传入）→ 使用 <img> 展示静态帧
          ELSE → 展示占位 UI："实时流在 Sprint 5 接入摄像头后启用"
          → 不抛错、不重试、不影响页面其他功能
```

```typescript
// ★V1.1 组件新增 prop
interface Props {
  deviceCode: string
  protocol?: 'webrtc' | 'flv' | 'hls'
  quality?: 'original' | 'low'
  showControls?: boolean
  overlayCanvas?: boolean
  // Sprint 0-4 实验室阶段：传入 MinIO 帧 URL，组件降级为静态图展示
  minioFrameUrl?: string
}
```

**关键约束：**
- 组件 `onUnmounted` 时必须释放 `session_token`，防止流媒体服务端资源泄漏
- 播放失败（错误码 2013 RTSP 源不可达）时展示离线占位图，不无限重试
- **★V1.1**：接收到 HTTP 501 时必须优雅降级，不向用户展示报错弹窗

> 协议适用场景见 §8.1 协议适配策略表。

### 8.3 CalibrationView 实验室阶段说明（★V1.1）

> **背景**：在线标定工具需要摄像头画面作为背景，以便用户在实际画面上绘制 ROI。
> 实验室阶段无摄像头，改用 MinIO 中的录像截帧替代。

**实验室阶段（Sprint 0-4）实现方式：**

```typescript
// CalibrationView.vue 中的帧获取逻辑
const frameUrl = ref<string>('')

onMounted(async () => {
  if (featureFlags.liveStreamEnabled) {
    // Sprint 5+：通过 VideoPlayer 获取实时流
    mode.value = 'live'
  } else {
    // Sprint 0-4：从 MinIO 获取最近一帧截图
    // 调用内部接口 GET /internal/scenes/{sceneId}/latest-frame
    // 该接口由 replay-service 从最近一次 replay_session 提取最后一帧
    const { data } = await api.getLatestReplayFrame(props.sceneId)
    frameUrl.value = data.frame_url  // MinIO 预签名 URL（15分钟有效期）
    mode.value = 'static'
  }
})
```

|------|---------|---------|---------|
| `live`（Sprint 5+） | WebRTC/FLV 实时流 | 实时 | 无 |
| `static`（Sprint 0-4） | MinIO 最近截帧 | 手动点击刷新按钮 | "标定使用录像截帧，实时流将在Sprint 5接入摄像头后启用" |

### 10.2 乐观锁冲突处理规范

`PUT` 场景配置时可能触发 2001 乐观锁冲突，前端统一处理方式：

```typescript
// 标准乐观锁冲突处理 Composable
export function useOptimisticUpdate() {
  async function updateWithConflictRetry<T>(
    fetchFn: () => Promise<T>,
    updateFn: (latest: T) => Promise<T>,
  ): Promise<T> {
    let latest = await fetchFn()
    for (let attempt = 0; attempt < 3; attempt++) {
      try {
        return await updateFn(latest)
      } catch (err: any) {
        if (err.code === 2001) {
          // 版本冲突：弹窗提示用户，重新获取最新数据
          await ElMessageBox.confirm(
            '其他用户已修改该配置，点击确定加载最新版本后继续编辑',
            '版本冲突',
            { type: 'warning' }
          )
          latest = await fetchFn()
        } else {
          throw err
        }
      }
    }
    throw new Error('多次版本冲突，请手动刷新页面')
  }
  return { updateWithConflictRetry }
}
```
