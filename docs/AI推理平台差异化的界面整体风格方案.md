## 方案一：深空工业风（DeepSpace Industrial）

### 1.1 设计理念
**"暗夜中的精准之眼"** —— 针对钢铁厂中控室7×24小时监控场景设计。深色背景减少眼部疲劳，高对比度强调告警信息，符合工业HMI（人机界面）标准。参考航天控制台与精密仪器界面美学，强调专业、沉稳、可信赖。

### 1.2 色彩系统

```css

### 1.3 布局特征

**三栏沉浸式布局**
- **左侧导航**：窄边栏（64px图标+悬浮文字），背景 `--bg-secondary`，选中项左侧发光条（2px `--primary-500`）
- **顶部状态栏**：高度48px，透明背景，仅底部1px分割线。右侧实时显示：GPU负载率、网络延迟、当前告警计数（红色徽章）
- **主内容区**：无圆角设计，卡片使用1px实线边框，强调工业设备的棱角感

**特殊区域适配**
- **视频播放区**：纯黑背景（#000000），周围2px `--border-accent` 发光边框，选中时呼吸灯效果（关键监控位）
- **告警弹窗**：从右侧滑入（非居中模态），背景半透明 `--bg-secondary` + 背景模糊，严重告警带红色顶部进度条（倒计时自动确认）

### 1.4 组件风格

| 组件 | 样式定义 |
|------|---------|
| **按钮** | 主按钮：填充`--primary-600`，文字白色，无圆角（直角或2px微圆角）。危险操作：描边红色+填充深红（#3D0000） |
| **卡片** | 背景`--bg-secondary`，1px边框`--border-primary`，无阴影。hover时边框变为`--primary-500`（发光效果） |
| **表格** | 表头背景`--bg-tertiary`，行hover背景`rgba(0,180,216,0.08)`。选中行左侧2px蓝色标记 |
| **表单** | Input背景`--bg-tertiary`，边框聚焦时蓝色发光（box-shadow: 0 0 0 2px rgba(0,180,216,0.2)） |
| **标签** | 场景状态使用"药丸"形状（全圆角），但边框粗细2px。CRITICAL状态添加脉冲动画（box-shadow脉冲） |

### 1.5 低代码编排器适配
- **画布背景**：网格点阵（dot grid #21262D），深色科技感
- **节点卡片**：深色背景，顶部色条区分类型（视频源-蓝、算法-紫、告警-红）
- **连线**：贝塞尔曲线，默认`--text-tertiary`，选中/激活时为`--primary-500`带流动动画（dasharray流动）

### 1.6 适用场景
- ✅ **中控室大屏监控**（7×24小时暗光环境）
- ✅ **夜班操作员**（减少蓝光刺激）
- ✅ **紧急情况处置**（高对比度告警显眼）
- ❌ **明亮办公室环境**（反光严重）
- ❌ **户外移动端**（可见度不足）

## 方案二：云境极简风（Cloud Lite）

### 2.1 设计理念
**"清晰无负担的智能"** —— 针对工艺工程师、设备主管等**非编程专业用户**设计的浅色高效界面。强调内容优先、呼吸感、降低认知负荷。参考Figma、Notion等现代SaaS工具，将复杂AI逻辑以清晰可视化的方式呈现。

### 2.2 色彩系统

```css

### 2.3 布局特征

**双栏弹性布局**
- **左侧导航**：宽边栏（240px），白色背景，图标+文字并排。选中项背景`--primary-50`，文字`--primary-600`，左侧4px圆角蓝色竖条
- **顶部操作栏**：高度56px，白色背景，底部无分割线（使用阴影shadow-sm）。左侧面包屑，右侧全局搜索框（圆角全幅）+ 用户头像
- **内容区**：大量留白（padding 24-32px），卡片圆角8px，微阴影（shadow: 0 1px 3px rgba(0,0,0,0.05)）

**信息密度策略**
- **列表页**：宽松行高（56px/行），头像+名称组合，减少纯文本列
- **详情页**：标签页（Tabs）组织信息，卡片式分块，关键数据使用"数据看板"（大字体+图标）突出

### 2.4 组件风格

| 组件 | 样式定义 |
|------|---------|
| **按钮** | 主按钮：填充`--primary-500`，圆角6px，文字白色，hover时轻微上浮（translateY(-1px)）。次要按钮：白色背景+灰色边框 |
| **卡片** | 纯白背景，圆角8px，细边框`--border-primary`，hover阴影加深。选中状态：蓝色边框+蓝色左侧4px条 |
| **表格** | 表头文字灰色加粗，行hover背景`--bg-tertiary`。操作列使用图标按钮（编辑、删除）而非文字链接 |
| **表单** | 输入框圆角6px，背景白色，边框`--border-primary`，聚焦时边框变蓝+外发光。标签左对齐，字体中等 weight（500） |
| **标签** | 圆角全角（999px），小字号（12px），粗体。优先级P1使用红色背景+白色文字，P4使用灰色背景 |

### 2.5 特殊场景适配

**视频播放区**
- 卡片容器，圆角8px，白色背景，视频区域16:9比例。
- **ROI标定模式**：Canvas叠加半透明蓝色区域（rgba(37, 99, 235, 0.15)），控制点使用8px圆点（蓝色边框+白色填充）

**低代码编排器**
- **画布背景**：点状网格（#E5E7EB），浅色清爽
- **节点卡片**：纯白背景，微阴影，圆角8px。不同颜色顶部条（2px）区分类型
- **属性面板**：右侧固定宽度（320px），白色背景，与画布区分明显，表单控件密度紧凑

**告警提示**
- 使用全局通知中心（右上角Toasts），非模态打断。严重告警使用红色边框+图标，普通提示蓝色左侧条。

### 2.6 适用场景
- ✅ **办公室配置工作**（明亮环境，长时间编辑场景配置）
- ✅ **非技术用户**（工艺工程师、设备主管使用低代码编排器）
- ✅ **移动端适配**（浅色在户外可见度更好）
- ❌ **暗光监控室**（过亮导致眼部疲劳）
- ❌ **7×24小时值守**（蓝光影响昼夜节律）

## 方案三：赛博熔流风（CyberFlow）

### 3.1 设计理念
**"数据与钢铁的交响"** —— 强调AI算法的科技感与工业数据的流动性。采用深色背景+霓虹渐变，适合**大屏展示、领导汇报、技术Demo**。参考Cyberpunk 2077、数据可视化大屏设计，突出系统的"智能"与"未来感"。

### 3.2 色彩系统

```css
:root {
  /* 主色调 - 熔流渐变 */
  --primary-500: #00F0FF;        /* 电光青 */
  --primary-600: #00C2CC;
  --secondary-500: #FF0055;       /* 熔岩洋红 */
  --accent-500: #FFD700;          /* 炼金黄 */
  
  /* 渐变定义 */
  --gradient-primary: linear-gradient(135deg, #00F0FF 0%, #0077FF 100%);
  --gradient-alarm: linear-gradient(90deg, #FF0055 0%, #FF5500 100%);
  --gradient-success: linear-gradient(90deg, #00FF88 0%, #00CC66 100%);
  
  /* 背景层级 - 暗夜紫黑 */
  --bg-primary: #050511;         /* 深渊蓝黑 */
  --bg-secondary: #0A0A1F;         /* 暗物质蓝 */
  --bg-tertiary: #12122E;          /* 星云紫 */
  --bg-elevated: #1A1A3E;          /* 悬浮层 */
  
  /* 玻璃拟态 */
  --glass-bg: rgba(10, 10, 31, 0.7);
  --glass-border: rgba(0, 240, 255, 0.2);
  --glass-blur: blur(12px);
  
  /* 文字 */
  --text-primary: #FFFFFF;
  --text-secondary: #A0A0B0;
  --text-glow: rgba(0, 240, 255, 0.8);
}
```

### 3.3 布局特征

**沉浸无框布局**
- **去边框化**：极少使用实线边框，依靠背景色层级（--bg-primary → --bg-secondary）区分区域
- **玻璃拟态导航**：顶部导航栏半透明`--glass-bg` + 背景模糊，底部1px发光边框`--glass-border`
- **浮动侧边栏**：左侧导航悬浮于内容之上（margin 16px），圆角16px，玻璃效果，可收起为图标条

**动态元素**
- **实时数据流**：告警图标、推理计数器带有微光脉冲动画（text-shadow脉冲）
- **背景装饰**：可选低透明度粒子动画（Canvas背景）或数据流线条（非干扰性）

### 3.4 组件风格

| 组件 | 样式定义 |
|------|---------|
| **按钮** | 主按钮：使用`--gradient-primary`，圆角8px，文字白色带轻微发光（text-shadow）。hover时渐变亮度增加+外发光 |
| **卡片** | 背景`--bg-secondary`，圆角12px，无实心边框，使用内发光（box-shadow: inset 0 1px 0 rgba(255,255,255,0.1)）。hover时顶部出现`--primary-500`光条 |
| **数据指标** | 大数字（32px+）使用渐变填充（`background-clip: text`），后缀单位使用`--text-secondary`。带趋势箭头（上升绿/下降红发光） |
| **表格** | 行之间使用1px`--bg-tertiary`分隔线，无垂直边框。hover行背景`rgba(0,240,255,0.05)`+左侧2px青色条 |
| **标签** | 霓虹风格：深色背景+彩色边框（alarm-critical使用红渐变边框），文字与边框同色，微发光 |

### 3.5 特殊场景适配

**视频播放区**
- 视频区域圆角12px，外框使用`--gradient-primary`作为边框（2px渐变边框或发光效果）
- **检测框叠加**：AI识别框使用青色（#00F0FF）半透明边框（2px），带轻微发光效果，关键目标使用洋红（#FF0055）高亮

**低代码编排器**
- **画布**：深色网格+动态背景（可选星空/数据流）
- **节点**：不同类型的节点使用不同霓虹色（视频-青、算法-紫、告警-红），节点间连线使用渐变色（青→紫），选中时连线流动动画（光点流动）

**告警系统**
- 严重告警触发全屏边缘红光闪烁（body box-shadow红色脉冲），配合声音提示
- 告警卡片使用`--gradient-alarm`顶部条，文字发光效果

**模型管理/Sandbox**
- **版本状态条**：使用渐变色条展示模型从Staging→Production的流程，发光点表示当前状态
- **对比模式**：生产模型使用青色，Sandbox模型使用洋红色，对比画面左右分屏，中间发光分隔线

### 3.6 动效规范
- **转场**：页面切换使用淡入+轻微缩放（scale 0.98→1），时长200ms
- **hover反馈**：卡片hover时轻微上浮（translateY -2px）+阴影/发光增强
- **实时数据**：数字变化时使用"滚动数字"动画（odometer效果），图表线条绘制动画

### 3.7 适用场景
- ✅ **指挥中心大屏展示**（视觉冲击力强，领导视察）
- ✅ **技术Demo/展会演示**（突出AI科技感）
- ✅ **夜间高光环境**（工业现场有补光的情况）
- ❌ **日常长时间操作**（视觉疲劳，干扰性强）
- ❌ **数据密集配置工作**（过度设计分散注意力）

## 实施建议

### 主题切换架构

建议采用**CSS Variables + Element Plus Theme**双轨方案：

```typescript
// src/stores/theme.ts
export const useThemeStore = defineStore('theme', () => {
  const currentTheme = ref<'industrial' | 'cloud' | 'cyber'>('industrial')
  
  const applyTheme = (theme: typeof currentTheme.value) => {
    document.documentElement.setAttribute('data-theme', theme)
    // Element Plus 主题色同步
    const elTheme = {
      industrial: { primary: '#00B4D8', danger: '#FF3333' },
      cloud: { primary: '#2563EB', danger: '#DC2626' },
      cyber: { primary: '#00F0FF', danger: '#FF0055' }
    }[theme]
    // 更新 Element Plus 全局配置
  }
  
  return { currentTheme, applyTheme }
})
```

### 环境适配策略

| 部署环境 | 推荐主题 | 理由 |
|---------|---------|------|
| **中控室/操作站** | 方案一：深空工业 | 暗光环境，7×24监控，高对比度告警 |
| **工程师办公室/配置端** | 方案二：云境极简 | 明亮环境，长时间配置，低认知负荷 |
| **展示大屏/领导驾驶舱** | 方案三：赛博熔流 | 视觉冲击，强调AI属性，展示用 |
| **移动端巡检** | 方案二：云境极简 | 户外可见度，触控友好 |

### 技术实现要点

1. **Element Plus 主题覆盖**：三套方案需分别覆盖El组件的CSS变量（--el-color-primary等）
2. **视频播放器适配**：视频区域需独立于主题（保持原始视频色彩），但控制栏跟随主题
3. **图表库同步**：ECharts图表需与当前主题色系统同步（使用useThemeStore动态更新color palette）
4. **低代码编排器**：vue-flow组件需深度自定义node/edge样式以匹配各主题风格

### 无障碍与可访问性

- **方案一**：确保对比度≥4.5:1（除次要文字），告警色考虑红绿色盲（使用图标+文字双重提示）
- **方案二**：标准WCAG 2.1 AA级对比度，支持系统级prefers-color-scheme自动切换
- **方案三**：虽为展示用途，但仍需保证关键数据可读性（避免过度发光导致文字模糊）

---

**推荐决策**：建议以**方案一（深空工业）**作为默认生产环境主题（符合工业现场实际需求），**方案二（云境极简）**作为管理配置端可选主题，**方案三（赛博熔流）**作为大屏专用主题，通过用户角色或访问入口自动切换。
