### 3.1 【高优先级】alarm-rule-service 单元测试

**对应需求**：M1-07 修复（`POST /models/{id}/promote` 路径）、状态机流转、四眼原则

**目标文件**：
```
services/alarm-rule-service/src/test/java/com/tianzhu/tianjing/alarm/service/
  ├── ModelVersionServiceTest.java   ← 状态机 + 四眼原则
  └── AlarmRuleServiceTest.java      ← 告警规则 CRUD + 优先级
```

**ModelVersionServiceTest 必须覆盖的用例**：

| 用例 | 验收标准 |
|---|---|
| 状态机合法流转：STAGING → SANDBOX_VALIDATING → REVIEWING → PRODUCTION | 每步流转成功 |
| 状态机非法跳步：STAGING 直接 → PRODUCTION | 抛出业务异常，返回错误码 |
| 四眼原则：提交人与审批人相同 | 返回 `4004` |
| 四眼原则：不同 MODEL_REVIEWER 审批 | 状态变更为 REVIEWING/PRODUCTION |
| promote 接口路径存在且可调通 | HTTP 200，触发蓝绿切换逻辑 |
| 蓝绿切换：旧 PRODUCTION 版本自动 DEPRECATED | 新版本上线后旧版本状态变更 |
| 覆盖率要求 | Service 层行覆盖率 ≥ 80% |
